# Лабораторная работа №5

## Запуск MPI-программы на суперкомпьютере «Сергей Королёв»

## Цель работы

Изучить основы параллельного программирования с использованием MPI, реализовать параллельное умножение матриц и выполнить запуск программы на суперкомпьютере «Сергей Королёв».

---

## Описание программы

Программа реализует параллельное умножение квадратных матриц с использованием MPI.

Каждый процесс получает часть строк матрицы A, после чего выполняет вычисление своей части результирующей матрицы C.

Основные этапы работы программы:

* инициализация MPI
* определение количества процессов
* распределение строк матрицы между процессами
* передача матрицы B всем процессам через `MPI_Bcast`
* вычисление локального блока результата
* сбор итоговой матрицы через `MPI_Gather`
* измерение времени выполнения с помощью `MPI_Wtime`

---

## Исходный код

Программа содержит следующие основные функции:

```c
double* create_matrix(int rows, int cols)
```

Выделение памяти под матрицу.

```c
void random_fill(double* mat, int n)
```

Заполнение матрицы случайными числами.

```c
void multiply_block(double* A, double* B, double* C, int n, int rows)
```

Вычисление части результирующей матрицы для текущего MPI-процесса.

### Используемые MPI-функции

| Функция       | Назначение                        |
| ------------- | --------------------------------- |
| MPI_Init      | Инициализация MPI                 |
| MPI_Comm_rank | Получение номера процесса         |
| MPI_Comm_size | Получение количества процессов    |
| MPI_Bcast     | Передача матрицы B всем процессам |
| MPI_Scatter   | Распределение строк матрицы A     |
| MPI_Barrier   | Синхронизация процессов           |
| MPI_Wtime     | Измерение времени выполнения      |
| MPI_Gather    | Сбор результирующей матрицы       |
| MPI_Finalize  | Завершение работы MPI             |

---

## Компиляция программы

```bash
mpicc main.c -o mpi_matrix
```

---

## Скрипт запуска Slurm

```bash
#!/bin/bash
#SBATCH --job-name=mpi_matrix
#SBATCH --time=00:10:00
#SBATCH --nodes=2
#SBATCH --ntasks-per-node=4
#SBATCH --partition=batch
#SBATCH --output=result.out

mpirun ./mpi_matrix
```

## Анализ результатов

Результаты показывают, что при увеличении количества процессов время выполнения программы существенно уменьшается.

Основные наблюдения:

* наибольшее ускорение наблюдается на матрицах большого размера
* при увеличении числа процессов вычислительная нагрузка распределяется между процессами более эффективно
* для небольших матриц влияние MPI менее заметно из-за накладных расходов на передачу данных и синхронизацию
* с ростом размера матриц эффективность распараллеливания увеличивается
* распределение строк матрицы между процессами обеспечивает равномерную загрузку вычислительных ресурсов

Для матрицы размером 2000×2000 время выполнения уменьшилось:

* с 118.406 сек на 1 процессе
* до 14.681 сек на 8 процессах

Для матрицы размером 1600×1600 время выполнения сократилось:

* с 59.738 сек на 1 процессе
* до 7.642 сек на 8 процессах

Полученные результаты подтверждают эффективность использования MPI для ускорения ресурсоёмких вычислений.
