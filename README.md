

# Лабораторная работа №2

## Параллельное умножение матриц с использованием OpenMP

---

## Задание

Модифицировать программу из л/р №1 для параллельной работы по технологии OpenMP.  Провести серию экспериментов с разным количеством потоков (1, 2, 4, 8 и т.д.), разными размерами матриц (примерно 200, 400, 800, 1200, 1600, 2000), с разным количеством вычислительных ядер при наличии технической возможности (1, 2, 4, 8 и т.д. ), иначе использовать фиксированное существующее количество вычислительных ядер, например 4.


---

## Ход работы

В ходе выполнения лабораторной работы последовательный алгоритм умножения матриц был адаптирован для параллельного выполнения.

Основные вычисления (тройной цикл) были распараллелены с использованием директивы OpenMP:

```id="7r1xzk"
#pragma omp parallel for collapse(2)
```

Такой подход позволяет распределить вычисления сразу по двум измерениям результирующей матрицы, что даёт более равномерную загрузку потоков.

Программа выполняет следующие этапы:

* генерация двух исходных матриц со случайными значениями
* выполнение их параллельного умножения
* измерение времени выполнения
* сохранение результатов в файл
* запись статистики в CSV-формате

Для проверки корректности вычислений использовался отдельный Python-скрипт с применением библиотеки NumPy.

## Анализ результатов

Анализ полученных данных показывает, что влияние многопоточности на производительность носит неоднозначный характер.

Основные наблюдения:

* Для небольших размеров матриц (200–400) параллелизация практически не даёт выигрыша
* В ряде случаев увеличение числа потоков приводит к росту времени выполнения
* Для более крупных матриц наблюдается небольшое ускорение, однако оно значительно ниже теоретического

Подобное поведение можно объяснить следующими факторами:

* накладные расходы на создание и синхронизацию потоков
* ограниченная пропускная способность оперативной памяти
* конкуренция потоков за кэш процессора
* неравномерное распределение нагрузки

Таким образом, эффективность параллелизации ограничивается не только вычислительными ресурсами, но и архитектурными особенностями системы.

## Вывод

В рамках лабораторной работы была успешно реализована параллельная версия алгоритма умножения матриц с использованием OpenMP.

В ходе экспериментов установлено, что:

* применение многопоточности не всегда приводит к ускорению
* эффективность зависит от размера задачи
* после определённого порога рост числа потоков не даёт заметного прироста производительности

В целом, OpenMP является удобным инструментом для распараллеливания вычислений, однако для достижения максимальной эффективности необходимо учитывать особенности конкретной задачи и аппаратной платформы.
